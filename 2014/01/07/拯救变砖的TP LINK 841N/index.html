拯救变砖的TP LINK 841N

发表于 2014-01-07   |  

放假回家了，刚回家几天就有好几件郁闷事！一个是发现多拨竟然失效了！目前只能单拨（继续研究），估计是因为之前10拨太嚣张被封杀了~~ 另一个是家里的电脑被我搞坏了，貌似是掉进去个铁片，短路了，然后2个电容爆掉了。懒得折腾了。

今天打算继续弄多拨的，寻思给841n刷个固件，就从论坛上下了一个个人制作的，整合了nwan和mwan3。直接在之前的openwrt里升级，升级之后果断变砖！症状就是电脑获取不到IP，IP是169开头的；没有无线；系统灯一直闪。reset无效。

本来都放弃了，淘宝上买了ttl，准备开拆了。没想到拆到一半卡住了，死活拆不开。又回头搞，竟然搞好了。下面说一下我的步骤，其中的方法可以每个都试一下。

1.（重要）既然获取不了IP，手动设置IP为192.168.1.101，网关192.168.1.1。

2.这时候发现ping不通，但是在刚接通电源的几秒内可以ping通。网上不少人也是这种现象。

可以有2个办法进安全模式（这个需要固件刷成功，如果刷坏了没有安全模式）：第一种是在通电的几秒钟可以ping通路由器,然后你去按reset. 我试了下死活都没有进去；第二种在通电等全亮后, 在sys灯两的一瞬间按reset看到sys灯闪烁后就算成功了。我用第二种方法成功了，系统灯闪的很快，而且一直可以ping通。

3.这时候就可以Ctrl+C停止ping，telnet 192.168.1.1刷固件了。

下一个HFS，打开HFS把官方固件拖进去。

开始菜单 运行CMD ，打开dos界面。输入telnet 192.168.1.1，显示用户名密码 输入 root admin 或者自己的密码登陆成功后cd /tmp切换到路由器的 tmp文件夹下，执行 wget http://192.168.1.101/wr841nv7.bin（加重为地址是你自己的IP地址和固件文件名）。执行 mtd -r write wr841nv7.bin firmware。一开始我刷的是原厂固件，但是提示我Failed to earse block，换了openwrt的就好了。一分钟左右，telnet 丢失主机。路由器自动重启，久违的TP界面就来了。

附其它方法：

1.30/30/30大法：30/30/30 硬复位【即插电稳定后,按复位键不松开30秒 不松开断电30秒 不松开再插电30秒】以下操作将会清除 NVRAM(非易失性存储器)[1]:

在插上电源后,稳定的情况下，按住复位键不松开30秒。然后，继续不松开复位键，拔掉电源，持续等30秒。最后，还是继续不松开复位键，插上电源，持续再等30秒后, 才松开.松开30秒后进行一次 拔插 电来结束这次 30/30/30 硬复位操作 - 让它正常开机。这些操作应该在每次升级固件之前和之后都做一次。

2.ttl线刷：这个我还没尝试过，等以后再坏了的吧。
