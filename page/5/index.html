本地搭建LNMP or LAMP方便WordPress调试

发表于 2013-01-17   |  

之前一直在另一台VPS各种折腾，数据毁了也不怕，可是你懂得，那速度慢了又慢，操作还不是很方便，突然想到可以在本地测试嘛。Google之，果然可以非常方便的实现。

严格来说并不是本地LNMP或者LAMP，因为根本用不到Linux，只是搭建一个网站环境方便程序调试罢了。如果说非要LNMP或者LAMP的话，那得用虚拟机了，远没有这个方便快捷。

目测有两个软件，一个是国外的XAMPP，另一个是国人写的APMServ，分别介绍下：

一、XAMPP

XAMPP 是一个易于安装且包含 MySQL、PHP 和 Perl 的 Apache 发行版。XAMPP 的确非常容易安装和使用：只需下载，解压缩，启动即可。主页：http://www.apachefriends.org/zh_cn/index.html

XAMPP的安装非常方便，而且支持Windows、Linux、Mac OS X、Solaris四个系统，下载绿色版（http://www.apachefriends.org/zh_cn/xampp-windows.html）解压到C盘xampp，运行一下setup_xampp.bat即可（不解压到C盘xampp可能导致不能正常运行）。当然也可以下载安装版的，安装版有一个简单的安装过程可以引导~

①启动：启动 Apache 和 MySQL：xampp\xampp_start.exe

停止 Apache 和 MySQL：xampp\xampp_stop.exe

启动 Apache：xampp\apache_start.bat

停止 Apache：xampp\apache_stop.bat

启动 MySQL：xampp\mysql_start.bat

停止 MySQL：xampp\mysql_stop.bat

启动 Mercury 邮件服务器：xampp\mercury_start.bat

设置 FileZilla FTP 服务器：xampp\filezilla_setup.bat

启动 FileZilla FTP 服务器：xampp\filezilla_start.bat

停止 FileZilla FTP 服务器：xampp\filezilla_stop.bat

②安装WordPress：将你需要的程序安装到xampp根目录下的htdocs中即可~

如要访问，在浏览器输入http://localhost即可访问。

③XAMPP的其它常用路径

网站根目录的默认路径：xampp\htdocsMYSQL

数据库默认路径：xampp\mysql\data

④附xampp的默认密码

1) MySQL: User: root Password:（空）

2) FileZilla FTP: User: newuser Password: wampp

User: anonymous Password: some@mail.net

3) Mercury:Postmaster: postmaster (postmaster@localhost)

Administrator: Admin (admin@localhost)

TestUser: newuser Password: wampp

4) WEBDAV:User: wampp Password: xampp

这个是国产的，支持国产！不过貌似很久没更新了，作者比较忙，最新版本是5.2.6,6.0还在开发中~

WordPress只能看到一个主题

发表于 2013-01-17   |  

如果打开主题选项，发现只能看到现在在用的这个主题，其他装的完全看不到，安装时候还提示“已安装”，你不妨也试试下面的方法：

编辑/usr/local/php/etc/php.ini，搜索scandir，删除即可（即删除禁用scandir函数）。删除之后就可以看到所有主题了，还能检测到主题升级！

别忘了重启一下PHP-FPM：

/etc/init.d/php-fpm restart

PHP解释scandir功能是Returns an array of files and directories from the directory.之前装的lnmp是可以正常显示的，貌似是PHP5.4之后默认禁用了。

关于nginx网站IPV6配置的一点心得

发表于 2013-01-12   |  

考试终于考完了，有时间捣鼓一下博客了~

教育网有个很大的好处就是ipv6~给博客开启ipv6访问最起码自己能用到嘛。

nginx开启ipv6首先需要有ipv6的模块，编译nginx时加上-with ipv6，在dns里添加AAAA记录指向ip地址即可。关键是在配置虚拟机文件。

很多人说conf文件要改成这种形式：

listen 80;    #此处监听所有ipv4地址
listen [::]:80;    #此处监听所有ipv6地址

但我自己改过之后，重启nginx提示错误，大体意思是ip地址重复。。。。。发现添加上具体ip就可以了，如下所示：

listen ip:80;    #此处监听所有ipv4地址
listen [ipv6的地址]:80;    #此处监听所有ipv6地址

WordPress最后一个分页404的解决

发表于 2012-12-29   |  

码盲啊，代码就是看不懂，正好这个寒假可以了解一下PHP、javascript等方面的书，要原创质量才上的去~！

无意中发现最后一个分页打不开，提示404错误。其他分页正常，好奇怪哦！

网上朋友分析重写的问题，改了也没解决。无意中发现了一篇文章，试过之后问题解决了！感觉这个问题是倡萌主题都有的~十分感谢苏说博客！

先转载过来了~直接说方法了。

打开目录下wp-includes\class-wp.php 这个文件，要找到function handle_404()这个函数，原代码如下：

function handle_404() {
global $wp_query;

// If we’ve already issued a 404, bail.
if ( is_404() )
return;

// Never 404 for the admin, robots, or if we found posts.
if ( is_admin() || is_robots() || $wp_query->posts ) {
status_header( 200 );
return;
}

// We will 404 for paged queries, as no posts were found.
if ( ! is_paged() ) {

// Don’t 404 for these queries if they matched an object.
if ( ( is_tag() || is_category() || is_tax() || is_author() || is_post_type_archive() ) && $wp_query->get_queried_object() ) {
status_header( 200 );
return;
}

// Don’t 404 for these queries either.
if ( is_home() || is_search() ) {
status_header( 200 );
return;
}
}

// Guess it’s time to 404.
$wp_query->set_404();
status_header( 404 );
nocache_headers();
}
将其替换为如下代码并删除 && !is_paged() 注意备份：

function handle_404() {
global $wp_query;
if ( !is_admin() && ( 0 == count( $wp_query->posts ) ) && !is_404() && !is_robots() && !is_search() && !is_home() ) {
// Don’t 404 for these queries if they matched an object.
if ( ( is_tag() || is_category() || is_tax() || is_author() ) && $wp_query->get_queried_object() && !is_paged() ) {
if ( !is_404() )
status_header( 200 );
return;
}
$wp_query->set_404();
status_header( 404 );
nocache_headers();
} elseif ( !is_404() ) {
status_header( 200 );
}
}

问题解决了~

phpmyadmin登陆后跳到首页&显示错误的解决

发表于 2012-12-27   |  

今天博客搬了家，打开wordpress和phpmyadmin后显示不全，一大片空白，只剩下文字，就像网速非常慢的样子，而且phpmyadmin不能登录，输入完用户名密码之后直接跳到博客首页~

分析了一下，Wordpress显示不全应该是wp super cache插件的问题，因为搬家后改了网站的路径。果断停止卸载，Wordpress恢复正常。

phpmyadmin的问题，感觉跟伪静态什么的有关系，也不太明白。想起来在编译lnmp是开启了伪静态并且选择了wordpress。遂修改nginx的配置文件。

server
    {

        server_name www.maitaidan.com maitaidan.com maitaidan.info www.maitaidan.info;

    include wordpress.conf;
    location ~ .*\.(php|php5)?$
        {
            try_files $uri =404;
            fastcgi_pass  unix:/tmp/php-cgi.sock;
            fastcgi_index index.php;

    location ~ .*\.(gif|jpg|jpeg|png|bmp|swf)$

这是有问题的，可以看到其中有一句include wordpress.conf，果断删掉~为了还能实现固定链接不出现404，再加上如下语句：

}

无需输入FTP信息就能升级 WordPress 及插件

这个问题网上有很多了，我权当记录一下了~

原因是跑PHP进程的用户名和WP文件夹的所有者不同。

进入WordPress目录，运行：

chown -R www *

所有者改为www搞定。
