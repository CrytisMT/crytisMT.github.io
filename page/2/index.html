Cannot refer to a non-final variable but inside an inner class defined in a different method

发表于 2014-02-09   |  

在使用Java局部内部类或者匿名内部类时，若该类调用了所在方法的局部变量，则该局部变量必须使用final关键字来修饰，否则将会出现编译错误“Cannot refer to a non-final variable * inside an inner class defined in a different method” 下面通过一段代码来演示和分析原因。

public class Example {

public static void main(String args[]) {
    doSomething();
}

private static void doSomething() {
    final String str1 = "Hello";
    // String str2 = "World!";
    // 创建一个方法里的局部内部类
    class Test {
        public void out() {
            System.out.println(str1);
            // System.out.println(str2);
        }
    }
    Test test = new Test();
    test.out();

}

}
上面代码若去掉第9行和第14行的注释符号，则第14行就会给出“Cannot refer to a non-final variable inside an inner class defined in a different method”这样的编译错误。原因如下：*在方法中定义的变量时局部变量，当方法返回时，局部变量(str1,str2)对应的栈就被回收了，当方法内部类去访问局部变量时就会发生错误。当在变量前加上final时，变量就不在是真的变量了，成了常量，这样在编译器进行编译时(即编译阶段)就会用变量的值来代替变量，这样就不会出现变量清除后，再访问变量的错误。

更简单的看，匿名内部类中的方法调用了其他方法中的局部变量，所以要变成常量。

拯救变砖的TP LINK 841N

发表于 2014-01-07   |  

放假回家了，刚回家几天就有好几件郁闷事！一个是发现多拨竟然失效了！目前只能单拨（继续研究），估计是因为之前10拨太嚣张被封杀了~~ 另一个是家里的电脑被我搞坏了，貌似是掉进去个铁片，短路了，然后2个电容爆掉了。懒得折腾了。

今天打算继续弄多拨的，寻思给841n刷个固件，就从论坛上下了一个个人制作的，整合了nwan和mwan3。直接在之前的openwrt里升级，升级之后果断变砖！症状就是电脑获取不到IP，IP是169开头的；没有无线；系统灯一直闪。reset无效。

本来都放弃了，淘宝上买了ttl，准备开拆了。没想到拆到一半卡住了，死活拆不开。又回头搞，竟然搞好了。下面说一下我的步骤，其中的方法可以每个都试一下。

1.（重要）既然获取不了IP，手动设置IP为192.168.1.101，网关192.168.1.1。

2.这时候发现ping不通，但是在刚接通电源的几秒内可以ping通。网上不少人也是这种现象。

可以有2个办法进安全模式（这个需要固件刷成功，如果刷坏了没有安全模式）：第一种是在通电的几秒钟可以ping通路由器,然后你去按reset. 我试了下死活都没有进去；第二种在通电等全亮后, 在sys灯两的一瞬间按reset看到sys灯闪烁后就算成功了。我用第二种方法成功了，系统灯闪的很快，而且一直可以ping通。

3.这时候就可以Ctrl+C停止ping，telnet 192.168.1.1刷固件了。

下一个HFS，打开HFS把官方固件拖进去。

开始菜单 运行CMD ，打开dos界面。输入telnet 192.168.1.1，显示用户名密码 输入 root admin 或者自己的密码登陆成功后cd /tmp切换到路由器的 tmp文件夹下，执行 wget http://192.168.1.101/wr841nv7.bin（加重为地址是你自己的IP地址和固件文件名）。执行 mtd -r write wr841nv7.bin firmware。一开始我刷的是原厂固件，但是提示我Failed to earse block，换了openwrt的就好了。一分钟左右，telnet 丢失主机。路由器自动重启，久违的TP界面就来了。

附其它方法：

1.30/30/30大法：30/30/30 硬复位【即插电稳定后,按复位键不松开30秒 不松开断电30秒 不松开再插电30秒】以下操作将会清除 NVRAM(非易失性存储器)[1]:

在插上电源后,稳定的情况下，按住复位键不松开30秒。然后，继续不松开复位键，拔掉电源，持续等30秒。最后，还是继续不松开复位键，插上电源，持续再等30秒后, 才松开.松开30秒后进行一次 拔插 电来结束这次 30/30/30 硬复位操作 - 让它正常开机。这些操作应该在每次升级固件之前和之后都做一次。

2.ttl线刷：这个我还没尝试过，等以后再坏了的吧。

笔试题好考字符串的。

public class ReverseString {

public static void main(String[] args) {
    // TODO Auto-generated method stub
    String oldString = "abcdefgh";
    convert1(oldString);
    convert2(oldString);
    convert3(oldString);
}

public static void convert3(String oldString) {
    // TODO Auto-generated method stub
    for (int i = oldString.length() - 1; i &gt;= 0; i--) {
        System.out.print(oldString.charAt(i));
    }
    System.out.println();
}

public static void convert2(String oldString) {
    // TODO Auto-generated method stub
    char[] arr = oldString.toCharArray();
    for (int i = arr.length - 1; i &gt;= 0; i--) {
        System.out.print(arr[i]);
    }
    System.out.println();
}

public static void convert1(String oldString) {//用了StringBuffer的reverse()方法
    // TODO Auto-generated method stub
    StringBuffer temp = new StringBuffer(oldString);
    System.out.println(temp.reverse());
}

}

批处理一键切换、修改IP

发表于 2013-06-11   |  

由于某些需要需要来回切换IP，之前找了几个修改IP的软件，总有些bug，其实一想，还不如用批处理来的快捷方便。下面提供2个批处理用来切换、修改IP。

1.这个批处理可以在两个或多个IP、掩码、网关之间选择切换，没有加入dns的修改，有需要的可以自行添加。代码都很简单，一看就能懂。

上个效果图：

@echo off
color 0a
title IP切换脚本 麦钛蛋www.maitaidan.com
:start
@echo —————修改切换IP by麦钛蛋——————
@echo 请选择ip:
@echo 1.校园网
@echo 2.台式机
@echo —————www.maitaidan.com——————
set /p sLocal=请选择:
if %sLocal%==1 goto home
if %sLocal%==2 goto office
cls
@echo 选择错误，请重新选择
goto start
:home
@echo 正在设置
netsh interface ip set address 本地连接 static 8.8.8.8 255.255.255.0 8.8.8.254 1
goto end
:office
@echo 正在设置网络连接为192.168.137.2
netsh interface ip set address 本地连接 static 192.168.137.2 255.255.255.0 192.168.137.1
:end
echo 设置完成，自动退出

2.第二个批处理可以智能在两个IP之间切换，如果当前是这个IP，则切换为另外一个。

 

@echo off &setlocal enabledelayedexpansion
title IP智能切换脚本 麦钛蛋www.maitaidan.com
color e
Rem ‘/========获取本机的IP地址(局域网)=========/
echo 正在获取本机的IP地址，请稍等…
for /f “tokens=3 skip=2 delims=: “ %%i in (‘nbtstat -n’) do (
set “IP_addr=%%i”
set IP_addr=!IP_addr:~1,-1!
goto :next
)

:next

for /f “tokens=2 delims=:” %%b in (‘ipconfig^|find /i “ip”‘) do set fsip=%%b
echo 当前IP：%fsip%
Set fsip=%fsip: =%

echo ——– ip修改中,请稍后… ———

Rem ‘/=========判断当前IP=============/
if %fsip%==192.168.137.2 (

Rem ‘/=========设为家中IP=============/
netsh interface ip set address 本地连接 static 8.8.8.8 255.255.255.0 8.8.8.254 1

) ELSE (

Rem ‘/=========设为单位IP=============/
netsh interface ip set address 本地连接 static 192.168.137.2 255.255.255.0 192.168.137.1 1

)

ECHO ——– ip修改完成 ——–
ECHO —— 麦钛蛋www.maitaidan.com ——
pause
两个都不错，哈哈。
