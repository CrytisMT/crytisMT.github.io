Shell编程syntax error near unexpected token `{错误解决

发表于 2012-11-27   |  

在VPS整了一个自动同步备份网站到Dropbox的脚本（传送门），并且添加了每天4点运行5点结束，可几天过去了，上Dropbox一看，还是那大小，那文件，貌似没变啊，随手动运行了一下脚本，提示syntax error near unexpected token `{。

原来这是因为脚本在Windows下写的，传上去后多了点东西。

Vi脚本就能看到，每行后面多了“^M”，就是这玩意作怪。解决办法：

1、vi -b backup.sh,手动删除每行后面的^M。我的脚本没几行，vi也没学明白，我最喜欢这种办法了~

2、用vi的替换功能。

1)vi -bbackup.sh

2)在命令编辑行（ 按ESC键 然后shift+:冒号）输入：%s/^M//g

注意：上述命令行中的“^M”符，不是“^”再加上“M”，而是由“Ctrl+v+M”键生成的。
3、一些linux版本有 dos2unix 程序，可以用来祛除^M。

4、.cat filename1 | tr -d “\r” > newfile 去掉^M生成一个新文件，还有sed命令等，凡是可以替换的命令都是可以用来新生成一个文件的。

在贴一下关于“^M”的知识：

long long ago….. 老式的电传打字机使用两个字符来另起新行。一个字符把滑动架移回首位 (称为回车，<CR>，ASCII码为0D)，另一个字符把纸上移一行 (称为换行, <LF>，ASCII码为0A)。当计算机问世以后，存储器曾经非常昂贵。有些人就认定没必要用两个字符来表示行尾。UNIX 开发者决定他们可以用 一个字符来表示行尾，Linux沿袭Unix，也是<LF>。Apple 开发者规定了用<CR>。开发 MS-DOS以及Windows 的那些家伙则决定沿用老式的<CR><LF>。
因为MS-DOS及Windows是回车＋换行来表示换行，因此在Linux下用Vim查看在Windows下用VC写的代码，行尾后的“^M”符号，表示的是符。
