phpmyadmin登陆后跳到首页&显示错误的解决

发表于 2012-12-27   |  

今天博客搬了家，打开wordpress和phpmyadmin后显示不全，一大片空白，只剩下文字，就像网速非常慢的样子，而且phpmyadmin不能登录，输入完用户名密码之后直接跳到博客首页~

分析了一下，Wordpress显示不全应该是wp super cache插件的问题，因为搬家后改了网站的路径。果断停止卸载，Wordpress恢复正常。

phpmyadmin的问题，感觉跟伪静态什么的有关系，也不太明白。想起来在编译lnmp是开启了伪静态并且选择了wordpress。遂修改nginx的配置文件。

server
    {
        listen       80;
        server_name www.maitaidan.com maitaidan.com maitaidan.info www.maitaidan.info;
        index index.html index.htm index.php default.html default.htm default.php;
        root  /home/wwwroot/maitaidan;

    include wordpress.conf;
    location ~ .*\.(php|php5)?$
        {
            try_files $uri =404;
            fastcgi_pass  unix:/tmp/php-cgi.sock;
            fastcgi_index index.php;
            include fcgi.conf;
        }

    location ~ .*\.(gif|jpg|jpeg|png|bmp|swf)$
        {
            expires      30d;
        }

    location ~ .*\.(js|css)?$
        {
            expires      12h;
        }

    access_log off;
}</pre>

这是有问题的，可以看到其中有一句include wordpress.conf，果断删掉~为了还能实现固定链接不出现404，再加上如下语句：

if (!-e $request_filename) {
rewrite ^([_0-9a-zA-Z-]+)?(/wp-.) $2 last;
rewrite ^([_0-9a-zA-Z-]+)?(/..php)$ $2 last;
rewrite ^ /index.php last;
}
